2.4 选择器优先级如何确定？

2.4.1 比较同一级别的个数，数量多的优先级高

important > 内联 > ID > 类 > 标签 | 伪类 | 属性选择 > 伪对象 > 通配符 > 继承

  1. 权值的大小跟选择器的类型和数量有关，样式的优先级跟样式的定义顺序有关
    这也就解释了为什么 11 个标签的定义会比不上 1 个类的定义，1 个类加 11 个标签会比不上 2 个类的权重高。
  2. 一个反例：256 个 class 类名选择器干掉一个 id 选择器实例页面
    原因：所有的类名(classes)都是以 8 字节字符串存储的，8 字节所能 hold 的最大值就是 255。当同时出现 256 个 class, 势必会越过其边缘，溢出到 id 区域。
