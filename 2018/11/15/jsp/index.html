web 服务器概念及类型

  • 外部服务器又称 www 服务器网站服务器站点服务器，从本质上说它就是一个软件系统
  • iis 服务器、apache 服务器、tomcat 服务器、weblogic 服务器、jboss 服务器

C/S、B/S 体系结构内容以及区别

  • CS 体系结构
    典型的两层架构，客户端/服务器端结构

  • BS 体系结构
    浏览器/服务器结构

  • 它们之间的区别

    硬件环境不同，CS 建立在专用网络上，BS 建立在广域网络上
    对安全要求不同，CS 相对固定，BS 可公开
    对程序架构不同，CS 更注重流程
    软件重用不同，CS 不可避免整体性的考虑，bs 相对独立
    系统维护不同，CS 维护难，BS 维护简单
    处理问题不同，CS 可以处理固定用户群，bs 相对较散
    用户接口不同，CS 多建立在 windows 平台上，BS 多建立在浏览器
    信息流不同，CS 一般是典型的中央集权式处理，交互性较低；BS 信息流可以变换，更像一个市场中心

静态网页

  1. 每个静态网页都有一个固定的地址
  2. 静态网页发布到服务器上后，无论是否被访问，都是一个独立的文件
  3. 静态网页的内容相对稳定，不含特殊代码
  4. 由于不需要通过数据库工作，因此静态网页的访问速度比较快

动态网页

  1. 动态网站可以实现交互功能，如用户注册信息发布
  2. 动态网页并不是独立存在于服务器的网页文件，需要浏览器发出请求时才反馈
  3. 动态网页中包含有服务器端脚本

jsp+javabean

  • JSP 实现页面的显示，Javabean 对象用来保存数据和实现商业逻辑

JSP + javabean + servlet

  • 控制器的角色由 servlet 来实现，视图的角色由 JSP 页面来实现，模型的角色由 javabean 来实现

JSP 内置对象类型及作用范围

  • application 的作用范围
    作用范围是从服务器开始运行，到服务器关闭

  • session 作用范围
    在客户端与服务端相连接开始，到连接中断为止

  • request 的作用范围
    在一个 jsp 页面向另一个 jsp 页面提出请求到请求完成之间，再完成请求后结束

  • page 作用范围
    作用范围是当前页

servlet 工作原理及生命周期

  • 工作原理
    当服务器接收到一个 HTTP 请求时，服务器会将请求交给 servlet 的容器，容器首先对请求的 url 进行解析，并根据 web.xml 文件找到相应的 servlet，同时将 request response 传递给 servlet。servlet 通过 request 对象获取客户端请求者，请求信息及其他信息，servlet 处理完请求后，会把所有需要返回的信息放入 response 对象中，并返回给客户端，servlet 一旦处理完请求，servlet 容器就会刷新 response 对象，并将控制权重新交给服务器

  • 生命周期
    从服务器开始运行时开始，然后不停的处理来自浏览器的请求，并通过服务器将响应结果返回给客户端，直到服务器停止运行，servlet 才会被清除

javabean 概念及作用

  • 概念
    是一种 Java 组件技术，是由 sun 公司提出来为了适应网络计算的组件结构

  • 作用
    将 html 和 Java 代码分离，方便维护
    可以将经常使用的代码抽象成一个 javabean 组建，方便调用

过滤器概念及处理过程

  • 过滤器是服务器与客户端请求与响应的中间层组件
  • 客户端通过 request 进入到 filterchain，然后通过 servlet 之后再返回到 filterchain，通过 response 返回到浏览器

监听器概念及创作步骤

  • 监听器是 servlet 规范中定义的一种特殊类，其对应者观察者模式，当事件发生时会自动触发该事件对应的 listener

  • httpsessionlistener 监听器，负责监听 HTTP 会话
    sessioncreated（httpsessionevent se）；sessiondestroyed

  • servletcontextlistener 监听器，用于监听 servletcontext 创建和删除
    contextdestroyed（servletcontexevent sce）；contextinitialized

  • servletresquestlistener 监听器
    requestdestroyed（servletrequestevent sre）；requedtinitialized（）

xml 文件定义

ajax 概念及工作原理

  • ajax 又称异步 javascript 和 xml，是一种 web 应用程序，客户机技术是一种创建交互式网页应用的网页开发技术。
  • 通过浏览器的 ajax 引擎（xmlhttprequest）对象向服务器发送一个请求，并接收服务器的响应数据，然后用 javascript 来操作 dom 到实现动态局部刷新

strust2 mvc 模式概念及开发环境配置

  • 是一个基于 mvc 设计模式的 web 应用框架，遵循 mvc 设计理念的开源 web 框架

  • 模型
    主要负责管理应用程序的数据，通过响应视图的请求和控制器的指令来更新数据，在这其中一般使用 javabean 和 ejb 来实现业务的逻辑操作

  • 视图
    主要用于应用程序中处理数据的显示，在这中主要有 JSP 页面和 actionform 两部分

  • 控制器
    主要负责接收用户的请求和数据，并判断应该将请求和数据交给哪个模型来处理，以及处理后的请求和数据，应该调用哪个视图来显示。action，servlet 是其中的主要控制器

hibernate orm 概念作用及开发环境配置

  • om 是对象关系映射，他将表与表之间的操作映射为对象与对象之间的操作，从而实现通过操作实体类来操作表的目的。
  • hibernate 是 orm 的映射工具，其作为一个持久层框架体现出了它的设计理念，还提供了高效的对象到关系型数据库的持久化服务

spring 模块组成内容

  • 核心模块
    主要组成部分是 branfactory 类，该类提供了 spring 框架的核心功能，采用了工厂模式的实现反转控制及依赖注入

  • aop 模块
    集成了面向切面编程的功能，为基于 spring 的应用系统提供了事务等管理服务

  • dao 模块
    提供数据库操作的模块代码，他是 jdbc 的抽象层

  • orm 映射模块

  • mvc 模块
    支持各种视图技术以及将请求参数绑定到业务对象中
  • web 模块
    提供了 web 应用程序的上下文，它提供了对框架的支持
